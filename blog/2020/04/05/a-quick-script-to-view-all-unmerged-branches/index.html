我们使用 git 作为 版本控制工具，极大的提高了效率，但是随着业务的增多和自身对于提交内容原子性的要求，往往会产生很多的分支，这就难免有时候，在发版的时候，某些分支被遗忘提交，造成功能丢失等问题。

因而如果保证分支多而且不忘记合并，是一个我们需要解决的问题。

  • 单纯靠人为挨个去看，肯定是不行的
